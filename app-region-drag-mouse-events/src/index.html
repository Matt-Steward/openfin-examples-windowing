Draggable areas and mouse Events

The purpose of this example is to show how setting a draggable area on a frameless window will affect mouse Events for nested and overlapping elements.

We will set the app's draggable area - the DOM element with an id of 'toolbar' - adding the CSS property of 

-webkit-app-region: drag;

This CSS makes the DIV into our window dragger, however, property prevents some of the standard mouse Events firing.

The initial state of the DOM Elements with mouse listeners is orange and they have the CSS property '-webkit-app-region: no-drag;' to allow them to receive mouse Events, even when nested in a 'drag' region . When they receive a 'mouseover' they turn red and a 'mouseleave' turns them blue.

You will see that dragging the mouse to any of the orange elements highlights them in red, including the furthest left div nested in the draggable top-bar. If, however, the mouse leaves the nested element via the grey, draggable, top bar the 'mouseleave' Event will be prevented and the element will not turn blue.

We counter this on the top right element by nesting it within a DIV with its CSS set to '-webkit-app-region: no-drag;' (the initially pink area) and padding of 4px. When the mouse leaves the orange DIV it must pass over the 4px border of the 'no-drag' region, thus allowing a mouse Event to be dispatched before it enters the mouse-event-cancelling 'drag' zone.
